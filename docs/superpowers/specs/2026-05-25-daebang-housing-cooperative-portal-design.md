# 대방동지역주택조합 홈페이지 설계 명세

작성일: 2026-05-25  
상태: 구현 계획 전 사용자 검토 대상

## 1. 목표

대방동지역주택조합 홈페이지는 조합을 소개하는 공개 랜딩페이지와 로그인 기반 조합 운영 포털을 결합한다. 일반 방문자에게는 밝고 환영받는 사업 이미지를 제공하고, 인증된 이용자에게는 정보공개, 회계·실적보고, 소통, 개인별 납부 및 환불 현황을 안전하게 제공한다.

## 2. 구현 단계 분리

기능 범위가 넓으므로 다음 단위로 나누어 구현한다.

1. **공개 랜딩페이지와 포털 셸**
   - Next.js, shadcn/ui, Pretendard 기반 UI 구축
   - 히어로, 공개 메뉴, 공지·자료실·사업정보 바로가기, 조합원 로그인 유도
   - 로그인 이후 역할별 대시보드의 화면 구조와 빈 상태까지 구현
2. **인증·권한·문서공개**
   - 정식조합원, 환불조합원, 환불완료자, 관리자 역할
   - 정보공개 및 회계·실적보고 게시, 승인, 검색, 열람·다운로드 이력
3. **수납·알림·커뮤니케이션**
   - 엑셀 업로드 기반 납부관리, 미납 대상 산출과 관리자 승인 발송
   - 알림톡/SMS 연동, 이슈 게시판, 투표·설문, 감사로그

첫 구현 계획은 1단계만 대상으로 삼는다. 2단계와 3단계는 운영·법률 확인과 데이터 양식 확정 뒤 별도 계획으로 진행한다.

### 2026-06-25 반영된 운영 기능

- 로그인 사용자는 공지사항, 조합뉴스, 자유게시판 게시글을 개인자료실의 `보관한 게시글`에 저장할 수 있다.
- 로그인 사용자는 공개자료 메뉴의 `등록 자료`와 `자료실 열기` 목록에서 문서를 개인자료실의 `내 보관함`에 저장할 수 있다.
- 개인자료실에서 문서를 열면 오른쪽 서랍 안이 아니라 전체 화면 PDF 뷰어로 열린다.
- 모든 PDF 뷰어는 `PDF만 크게` 보기로 전환할 수 있으며, 필요 시 상세 정보 보기로 돌아갈 수 있다.
- 관리자 PDF 업로드는 기본값을 `자동 최적화`로 두고, 5MB 초과 PDF가 15% 이상 줄어드는 경우에만 최적화본을 최종 파일로 저장한다.
- 서버 저장 용량을 줄이기 위해 PDF 원본과 최적화본을 동시에 보관하지 않는다.

## 4. 메뉴 구조

### 공개 메뉴

- `조합소개`: 인사말, 조직도, 찾아오시는 길
- `사업정보`: 사업개요, 추진경과 요약, 조감도, 위치도, 단지계획
- `조합소식`: 외부 공개가 허용된 공지와 일정
- `자료실`: 보도자료, 관련 법령, 일반 안내자료
- `로그인`: 조합원·환불조합원 인증 진입

### 로그인 메뉴

- `정보공개`: 의무공개 자료, 총회·이사회, 계약·행정·법률 문서
- `회계·실적보고`: 예산·결산, 수입·지출, 자금집행, 감사보고, 추진실적
- `조합소식`: 권한별 공지와 중요알림
- `이슈의 장`: 정식 조합원만 이용하는 현안 토론, 댓글, 공식답변
- `투표·설문`: 정식 조합원 의견수렴과 투표
- `내 분담금`: 정식 조합원의 개인별 납부·미납·독촉 이력
- `내 환불현황`: 환불조합원의 정산 통지와 처리현황

### 자료실 구분

| 자료 유형 | 노출 범위 |
|---|---|
| 보도자료, 관련 법령, 일반 안내자료 | 일반 공개 |
| 조합규약, 내부 운영규정, 조합원용 서식 | 로그인 사용자 |
| 개인정보 포함 자료, 회계 원문, 계약·총회 문서 | 정보공개 또는 회계·실적보고 권한으로만 제공 |

로그인 사용자는 공개자료와 권한 내 문서를 개인자료실에 보관할 수 있다. 보관은 개인 편의 기능이며 문서의 공개 범위, 승인 상태, 감사 이력을 변경하지 않는다.

## 5. 랜딩페이지 화면 설계

### 5.1 로그인 전 인트로

인트로는 신길5동 사이트의 로그인·공지 접근성과 군포10구역 사이트의 사업 비주얼·추진자료 진입 구조를 참고하되, 오래된 조합 사이트 느낌 대신 밝고 현대적인 카드형 화면으로 만든다.

1. 고정 내비게이션: 로고, 공개 메뉴, `조합원 로그인` 보조 버튼, `사업정보 보기` 주 버튼
2. 히어로 섹션: 중앙 카피와 CTA, 주변에 주민·아파트 일러스트
3. 주요 바로가기 카드: 사업정보, 추진현황, 자료실, 조합소식
4. 일반 공지 카드: 공개 가능 공지 최신글
5. 사업현황 안내: 공개 가능한 추진 단계와 위치·조감도 링크
6. 조합원 전용 안내 패널: 정보공개, 회계보고, 분담금, 이슈 참여는 로그인 후 이용
7. 연락처 및 푸터: 사무실 정보, 오시는 길, 개인정보처리방침, 이용약관

## 8. 정보공개 및 운영 흐름

### 문서공개

1. 관리자가 유형, 작성·변경일, 공개 대상, 첨부파일을 등록한다.
2. 시스템은 공개 마감일과 승인 대기 상태를 관리한다.
3. 담당자 검토 및 중요자료 승인 후 게시한다.
4. 사용자 권한에 따라 열람하고, 다운로드·열람 이력을 남긴다.
5. 수정 시 기존 파일을 보존하고 버전과 수정사유를 기록한다.
6. 사용자는 필요한 문서를 개인자료실에 보관해 다시 열람할 수 있다.
7. PDF 열람 화면은 일반 상세 보기와 `PDF만 크게` 보기를 모두 제공한다.

PDF 업로드는 서버 저장 용량을 줄이기 위해 단일 최종 파일 저장을 원칙으로 한다. 기본값은 `자동 최적화`이며, 5MB 초과 PDF가 15% 이상 줄어드는 경우에만 최적화본을 저장한다. 도면, 직인, 서명, 스캔 품질이 중요한 문서는 관리자가 `원본 그대로 저장`을 선택할 수 있다.

### 납부와 미납 안내

1. 회계담당자가 표준 엑셀 양식으로 개인별 납부자료를 업로드한다.
2. 회원번호, 회차, 금액, 날짜, 중복, 권한 상태를 검증한다.
3. 관리자 승인 뒤 정식 조합원 본인 화면에 반영한다.
4. 미납 대상과 알림 문안은 시스템이 생성한다.
5. **관리자 승인 후에만** 알림톡 또는 SMS를 발송한다.
6. 미납 사실은 본인과 권한 있는 담당자에게만 노출한다.

### 알림과 커뮤니케이션

- 중요공지, 정보공개, 회계보고, 투표는 대상 권한별 알림톡/SMS 발송을 지원한다.
- 환불·정산 및 미납 안내는 개인 발송만 허용한다.
- 일반 카카오톡 단체채팅방 자동 게시를 전제로 삼지 않으며, 관리자 확인용 공유 문안을 생성한다.

## 9. 법률 및 개통 전 확인

새 홈페이지만을 공식 정보공개 창구로 사용할 계획은 구현과 별개로 확인이 필요하다. 개통 전에 다음 사항을 동작구·서울시 담당부서 또는 법률자문으로 서면 확인한다.

1. 로그인 기반 홈페이지 열람 방식이 주택조합 정보공개의 인터넷 공개 요건을 충족하는지
2. 서울시 정비사업 정보몽땅 병행 등록을 종료할 수 있는지
3. 환불 완료 전 환불조합원의 전체 또는 일부 자료 열람 범위가 적법한지
4. 분담금·미납·환불자료와 알림톡/SMS 발송의 개인정보 처리·위탁·증빙 요건

확인 결과가 현재 운영 구상과 다르면 권한 정책과 공개 동작을 개통 전에 조정한다.

## 10. 기술 방향

- 프레임워크: Next.js App Router, TypeScript
- 스타일: Tailwind CSS, shadcn/ui 기반 컴포넌트
- 폰트: Pretendard
- 이미지: Next.js `Image`, 프로젝트 `public/assets`에 선별 에셋 저장
- 모션: CSS keyframes 우선, 필요한 작은 상호작용만 모션 라이브러리 검토
- 품질: 반응형 레이아웃, 키보드 접근성, reduced-motion 대응, 이미지 최적화

첫 구현 사이클은 백엔드 인증·문서 DB·발송 연동을 만들지 않고, 추후 기능을 연결할 수 있는 공개 페이지와 역할별 포털 화면 구조를 완성한다.
