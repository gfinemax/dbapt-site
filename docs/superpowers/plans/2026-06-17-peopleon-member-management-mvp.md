# PeopleOn 조합원 관리 MVP 구현 계획

## 작업 순서

1. 매칭/통계 유틸 테스트를 먼저 작성한다.
2. PeopleOn 행과 홈페이지 `User` 계정을 비교하는 순수 유틸을 구현한다.
3. PeopleOn API를 서버에서 페이지 단위로 읽는 어댑터를 추가한다.
4. `/portal/admin/members` 서버 페이지와 대시보드 컴포넌트를 추가한다.
5. 기존 관리자 포털에 새 조합원 관리 페이지 링크를 추가한다.
6. UI 리뷰 기록과 검증 결과를 `_workspace`에 남긴다.
7. 후속 요청에 따라 `role`과 별도의 `memberType`을 추가해 예비조합원을 `MEMBER` 권한 안에서 구분한다.
8. 가입 승인 회원 자격 변경 관리 표와 PeopleOn 확인 필요 조합원 표에 `자격 구분` 컬럼을 추가한다.

## 구현 경계

- 변경 대상은 관리자 로그인 영역에 한정한다.
- PeopleOn API는 읽기 전용으로만 호출한다.
- API 키는 서버 환경 변수로만 사용하고 클라이언트에 전달하지 않는다.
- PeopleOn 호출 실패는 운영 안내 상태로 처리한다.
- 자격 변경 액션은 이번 페이지 안에서 새로 만들지 않고 기존 관리자 포털의 승인/전환 흐름으로 이어지게 한다.
- 예비조합원은 홈페이지 접근 권한상 `MEMBER`로 취급하고, 표시/통계/비교용 구분은 `memberType=PRELIMINARY`로 처리한다.
